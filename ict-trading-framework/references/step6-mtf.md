# Step 6: Multi-Timeframe (MTF)

## Timeframe Roles

| TF | Role | Purpose |
|----|------|---------|
| HTF (Daily/4H) | Bias + Target | Direction and liquidity |
| MTF (1H/15M) | Confirmation | CISD/MSS |
| LTF (5M/1M) | Execution | FVG entry |

## HTF Roles

- Define major liquidity pools (PWH/PWL, PMH/PML)
- Establish one-sided bias using closures
- Identify swing points
- Determine WHERE price is heading

## MTF Roles

- Confirm CISD at HTF swing points
- Identify manipulation legs
- Bridge HTF narrative to LTF execution

## LTF Roles

- Entry refinement
- Precise execution
- Risk management

## Alignment Rules

- All 3 TFs must point same direction for highest probability
- HTF bias bullish + LTF bullish = aligned
- HTF bullish + LTF bearish = wait or no trade
- NEVER trade against HTF direction

## Decision Logic

1. Mark HTF liquidity (PDH/PDL, PWH/PWL)
2. Check HTF bias (continuation vs reversal)
3. Drop to MTF, find CISD in HTF direction
4. Drop to LTF, find entry at FVG
5. Execute only if all TFs aligned

## Works When

- HTF bias clear
- MTF confirms with CISD/MSS
- LTF entry aligns
- All 3 TFs pointing same way

## Fails When

- HTF unclear or ranging
- MTF conflicts with HTF
- LTF entry without HTF alignment

## No Trade Conditions

- HTF consolidation
- HTF and LTF conflict
- Missing any TF confirmation

## Note

- HTF direction > LTF signal
- Skip LTF if HTF unclear
- When in doubt, stay in HTF direction
